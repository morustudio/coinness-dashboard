# 코인니스 BTC 선물 지표 수집 시스템

## 프로젝트 개요

Coinness Futures Dashboard — 바이낸스 선물 기반 BTC 지표를 자동 수집하고 GitHub Pages로 시각화하는 시스템.

- **대시보드**: https://morustudio.github.io/coinness-dashboard/
- **저장소**: https://github.com/morustudio/coinness-dashboard
- **로컬 폴더**: `C:\Users\Administrator\Downloads\crawler\git_coinness`

---

## 구성 파일

| 파일 | 역할 |
|------|------|
| `crawler_v2.py` | 지표 수집기 (로컬 PC 실행) |
| `dashboard_v2.html` | 대시보드 (GitHub Pages 서빙) |
| `funding_data_v2.csv` | 수집 데이터 누적 저장 |
| `index.html` | GitHub Pages 진입점 |
| `crawler_v2.log` | 수집 로그 (로컬만, .gitignore) |

---

## 자동화 파이프라인

```
1. 수집  — crawler_v2.py 로컬 실행 (스케줄 자동)
2. 저장  — funding_data_v2.csv 로컬 append
3. 푸시  — git push → GitHub 자동 업로드
4. 서빙  — GitHub Pages → CSV 공개 URL
5. 갱신  — 대시보드 5분마다 자동 fetch + 표시
```

## crawler_v2.py — 수집기

### 버전 히스토리
- **v2.0** — 최초 작성 (1행/수집, 21회/일)
- **v2.1** — 정산 기준 4시간대 수집, 항목별 재시도
- **v2.002** — 김치프리미엄 수집 추가 (업비트 KRW + 환율 API)
- **v2.003** — 테이커 매수비율 + 오더북 매수/매도벽 수집 추가 (바이낸스 공개 API, 각 1개씩 총 4컬럼)

### 수집 스케줄 (21회/일)
- **정산 전후**: 00:00 / 00:30 / 01:00 / 01:30 / 02:00
- **정산 전후**: 08:00 / 08:30 / 09:00 / 09:30 / 10:00
- **정산 전후**: 16:00 / 16:30 / 17:00 / 17:30 / 18:00
- **일반**: 04:00 / 06:00 / 12:00 / 14:00 / 20:00 / 22:00

### 수집 항목
- 롱숏비율 (1h / 4h / 12h / 24h)
- 강제청산 금액 / 비율
- 펀딩비율 (바이낸스 / OKX / 바이비트)
- 미결제약정 (바이낸스)
- BTC 현재가 (코인니스)
- 김치프리미엄 (업비트 + 환율)
- 테이커 매수비율 (바이낸스 선물, 5분 기준)
- 오더북 매수벽 / 매도벽 (바이낸스 선물, 상위 500호가 중 최대 버킷 1개씩)

### 실행 명령어
```powershell
py crawler_v2.py --test   # 즉시 1회 테스트
py crawler_v2.py          # 스케줄러 가동
```

### 버전 히스토리

#### v2.004

- 시간대 버튼 클릭 기능 수정

#### 수익 계산기 — 수익 탭
- 기준: 바이낸스 선물 BTCUSDT
- 메이커(0.02%) / 테이커(0.05%) 선택
- 롱 / 숏 포지션 선택
- 레버리지 1x~100x
- 오픈가 / 청산가 / 수량 입력
- 출력: 총수익 / 총수수료 / 최종수익 / 수익률

#### v2.005 (2026-05-17)
- 차트 기본값: 전체 → 1일로 변경
- 수익 계산기에 **R:R 전략 탭** 추가
  - 입력: 진입가 / 손절% / 1차 익절% / 2차 익절%
  - 출력: 1차·2차 익절가 / 손절가 / 청산가(강제) / R:R 비율
  - 계산 공식 (수수료 반영):
    - 롱 익절가 = 진입가 × (1 + 목표%/레버리지/100 - 수수료×2)
    - 숏 익절가 = 진입가 × (1 - 목표%/레버리지/100 + 수수료×2)
    - 청산가(롱) = 진입가 × (1 - 1/레버리지)
    - 청산가(숏) = 진입가 × (1 + 1/레버리지)
- AI 분석 프롬프트 강화
  - 지표 우선순위: OI → 청산 → 펀딩비 → 김프 → 롱숏비율
  - 확률 채점표 5항목 × 20점
  - 펀딩비 정산 30분 전 타이밍 패턴 판단 기준 추가
- 프롬프트 시간대 태그 추가: `[05-17 09:00][1시간]`, `[4시간]`, `[12시간]`, `[24시간]`
